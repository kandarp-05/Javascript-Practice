callback and promises
promises are special js objects that links the producing code and consuming code together.
producing code may take its time to give the result but it will give it to 
consumers
let promise=new Promise(function(resolve,reject){
    ...
});
the function passed to the new promise is called the executor.
the executor obtains result then call either resolve or reject.
if job finished -> resolve(value) , a value is returned;
if job failed -> reject(error), error is returned;



                          {state:fulfilled
                          result:done}
                        /
executor {state:pending/
        result:undefined}
                       \
                        \ 
                          {state: rejected
                          result:error}

                          successfully completely
let promise = new Promise(function(resolve, reject) {
  // the function is executed automatically when the promise is constructed

  // after 1 second signal that the job is done with the result "done"
  setTimeout(() => resolve("done"), 1000);
});
state: pending->fulfilled
result: undefined->done

                          failed to fulfill

let promise = new Promise(function(resolve, reject) {
  // after 1 second signal that the job is finished with an error
  setTimeout(() => reject(new Error("Whoops!")), 1000);
});
state:pending->rejected
result:undefined->error

consuming functions can be registered using .then and .catch

syntax:
promise.then(
    function(result){...}
    ,function(error){...}
)
1st arg run when promise is resolved , 2nd will run when promise is rejected

.catch() is used when we are only interested in handling errors , here we 
pass null as the 1st arg

.finally() always run when the the promise is settled, either resolved or rejected.
